hexo使用备忘录

hexo搭建网站一些备忘记录

常见命令

  • hexo init 初始化本地文件夹为网站的根目录
  • hexo new 新建一篇文章，可简写hexo n
  • hexo generate 生成静态文件，简写hexo g
  • hexo server 启动本地服务器,简写hexo s
    • -p选项，指定服务器端口，默认4000
    • -i选项，指定服务器IP地址，默认0.0.0.0
    • -s选项，静态模式，仅提供public文件夹中的文件禁用文件监视
    • 运行服务器前需要安装hexo-server插件npm install hexo-server --save
  • hexo deploy 部署网站
  • hexo clean 清理缓存文件
  • Option
    • hexo –safe 安全模式，禁用加载插件和脚本
    • hexo –debug 调试模式，将消息详细记录到终端和debug.log文件
    • hexo –silient 静默模式

写文章流程

  1. 在hexo主目录输入命令:`hexo new “title”
  2. 在/hexo/source/_posts/目录下找到title.md文件，并编辑文章。
  3. 返回hexo主目录输入命令:hexo g生成静态文件。
  4. 把文件夹public里面的静态文件拷贝到服务器的网页根目录。
    • 我使用的方法是在本地public内使用git push gitbub，再在服务器的网页根目录使git pull到服务器本地。但是遇到个麻烦事，当我使用hexo clean的时候，会把public目录的.git删除掉。只能从来git clone，好麻烦的说。暂时没有找到好的方法解决这个问题。
